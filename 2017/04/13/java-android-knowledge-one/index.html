java 装箱和拆箱知多少

发表于 2017-04-13

java数据类型

java语言是对象的天下，对象通过new关键字创建，并保存在堆中，通过堆栈中的引用使用这些对象，需要占用一定的空间。对于常用的数据类型，这么做未免太复杂，所以产生了常用的基本类型。

下图展示了基本数据类型，以及占用空间大小，优于和机器硬件架构无关，所以可移植性比较强。

为何有包装数据类型？
java是面向对象的语言，为了和其他对象兼容，产生了包装类型，比如在Collection时会使用这些包装类型。

java装箱和拆箱

这里说的装箱和拆箱是针对基本类型到包装类型的转换.
比如:

Integer a = 100; 自动装箱
int b = a;  自动拆箱

    
Integer b = new Integer(100)
这个类型没有设计到类型转换，不能称之为装箱。

请看一下题目：

public void m21() {
            Integer a = new Integer(100);
            Integer b = new Integer(100);
            System.out.println("m21 result " + (a == b));
        }
        public void m22() {
            Integer a = new Integer(128);
            Integer b = new Integer(128);
            System.out.println("m22 result " + (a == b));
        }
       public void m31() {
            Integer a = 100;
            Integer b = 100;
            System.out.println("m31 result " + (a == b));
        }
        public void m32() {
            Integer a = 128;
            Integer b = 128;
            System.out.println("m32 result " + (a == b));
        }
       public void m41() {
            Integer a = new Integer(100);
            Integer b = new Integer(100);
            System.out.println("m41 result " + (a.equals(b)));
        }

输出结果是：

m21 result false
m22 result false
m31 result true
m32 result false
m41 result true

为什么m31会输出true？，其实在Integer类自动装箱时，做了一个缓存处理
